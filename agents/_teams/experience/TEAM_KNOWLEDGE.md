# Experience Team - 공유 지식

> 사용자 리서치, UX/UI 디자인을 담당하는 경험팀의 공유 지식 베이스

## Team Mission

사용자의 진짜 니즈를 발견하고, 직관적이고 매력적인 경험으로 설계합니다.

---

## 공유 프레임워크

### 1. 디자인씽킹 프로세스

```
공감 (Empathize) → 정의 (Define) → 아이디어 (Ideate) → 프로토타입 (Prototype) → 테스트 (Test)
```

#### 공감 (Empathize)
- 관찰: 사용자 행동 직접 관찰
- 인터뷰: 깊이 있는 1:1 대화
- 몰입: 사용자 입장에서 경험

#### 정의 (Define)
- 인사이트 도출
- POV (Point of View) 작성
- How Might We 질문

#### 아이디어 (Ideate)
- 브레인스토밍
- 아이디어 선별
- 컨셉 발전

#### 프로토타입 (Prototype)
- Lo-fi 프로토타입
- Hi-fi 프로토타입
- 실험 설계

#### 테스트 (Test)
- 사용성 테스트
- 피드백 수집
- 반복

### 2. 사용자 리서치 방법론

#### 정성 리서치
| 방법 | 언제 사용 | 아웃풋 |
|------|----------|--------|
| 심층 인터뷰 | 동기, 맥락 이해 | 인사이트, 인용문 |
| 맥락적 탐구 | 실제 환경 이해 | 행동 패턴, 워크플로우 |
| 다이어리 스터디 | 장기적 행동 파악 | 습관, 시간별 변화 |
| 포커스 그룹 | 다양한 관점 수집 | 의견 범위, 토론 |

#### 정량 리서치
| 방법 | 언제 사용 | 아웃풋 |
|------|----------|--------|
| 설문조사 | 대규모 의견 수집 | 통계, 세그먼트별 차이 |
| A/B 테스트 | 디자인 안 비교 | 전환율, 선호도 |
| 분석 데이터 | 행동 패턴 파악 | 퍼널, 이탈 지점 |
| 카드소팅 | 정보구조 설계 | IA, 메뉴 구조 |

## 팀 협업 프로토콜

### Strategy 팀과 협업
- **Input 수령**: 타겟 사용자 정의, 검증할 가설, 핵심 기능
- **Output 제공**: 페르소나, 여정 맵, 사용자 니즈 인사이트

### Growth 팀과 협업
- **Input 수령**: 콘텐츠 반응 데이터, A/B 테스트 결과
- **Output 제공**: 사용자 언어, 감정 트리거, 페인포인트

### Engineering 팀과 협업
- **Input 제공**: 화면 설계, 인터랙션 명세, 디자인 토큰
- **Output 수령**: 기술적 제약, 구현 가능성 피드백

## 의사결정 원칙

### 1. 사용자 증거 기반
```
"우리가 본 것" > "우리가 생각하는 것"
관찰, 인터뷰, 데이터로 뒷받침
```

### 2. 공감 우선
```
판단하지 않고 이해
"왜 그렇게 행동할까?"에 집중
```

### 3. 반복과 개선
```
완벽한 디자인은 없음
테스트 → 학습 → 개선 사이클
```

## 공유 용어집

| 용어 | 정의 |
|------|------|
| UX (User Experience) | 제품 사용 전체 경험 |
| UI (User Interface) | 사용자와 시스템 접점 |
| IA (Information Architecture) | 정보 구조 설계 |
| Wireframe | 레이아웃 스케치 |
| Mockup | 시각적 디자인 완성본 |
| Prototype | 인터랙션 포함 시뮬레이션 |
| Affordance | 사용 방법을 암시하는 특성 |
| Mental Model | 사용자의 기대 모델 |
| Pain Point | 사용자가 겪는 불편/고통 |
| Aha Moment | 가치를 인지하는 순간 |
| Friction | 사용 흐름의 마찰/장애물 |
| Accessibility | 접근성 (a11y) |
