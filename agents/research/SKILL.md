# Research Agent

> 시장, 경쟁사, 타겟 유저를 깊이 이해하여 전략의 기반을 마련합니다.

## Trigger

- Orchestrator에서 Research 단계로 라우팅될 때
- "리서치 해줘", "시장 조사", "경쟁사 분석", "유저 리서치"
- "타겟 분석", "페르소나 만들어줘"

## Input

```yaml
required:
  - prd: PRD 문서 (목표, 타겟 사용자, 문제 정의)
  
optional:
  - references: 참고 URL, 문서
  - constraints: 리서치 범위 제한
  - focus_areas: 특별히 깊이 조사할 영역
```

---

## Process

### Step 1: 리서치 범위 정의

PRD를 기반으로 리서치 질문을 도출합니다.

```markdown
## 리서치 질문 프레임워크

### 시장 (Market)
- 이 시장의 규모는? (TAM → SAM → SOM)
- 시장 트렌드와 성장률은?
- 규제나 진입 장벽이 있는가?

### 경쟁 (Competition)
- 직접 경쟁자는 누구인가?
- 간접 경쟁자/대안은 무엇인가?
- 경쟁자의 강점과 약점은?
- 미충족 니즈(Gap)는 어디에 있는가?

### 고객 (Customer)
- 타겟 고객은 정확히 누구인가?
- 그들의 Pain Point는 무엇인가?
- 현재 어떻게 문제를 해결하고 있는가?
- 얼마나 절실하게 해결책을 원하는가?
```

### Step 2: 데스크 리서치 실행

**시장 조사**
```
web_search 활용:
- "[시장명] market size 2024"
- "[시장명] trends report"
- "[키워드] industry analysis"
```

**경쟁사 분석**
```
각 경쟁사별:
- 서비스 URL 방문 (web_fetch)
- 핵심 기능 매핑
- 가격 정책 파악
- 사용자 리뷰 분석 (앱스토어, 커뮤니티)
```

**고객 인사이트**
```
커뮤니티/포럼 조사:
- Reddit, 블라인드, 커뮤니티 검색
- 관련 키워드로 Pain Point 수집
- "왜 [기존 서비스]가 싫은가" 유형의 불만 수집
```

### Step 3: 인사이트 종합

수집한 데이터를 구조화하여 핵심 인사이트를 도출합니다.

```markdown
## 인사이트 도출 프레임워크

### 기회 영역 (Opportunities)
- 시장에서 발견한 Gap은?
- 경쟁사가 놓치고 있는 것은?
- 고객이 가장 절실하게 원하는 것은?

### 위험 요소 (Risks)
- 진입 장벽은?
- 강력한 경쟁자의 대응 가능성은?
- 시장 변화 가능성은?

### 전략적 시사점 (Implications)
- 우리의 포지셔닝은?
- 최초 타겟 세그먼트는?
- 차별화 포인트는?
```

## Quality Checklist

산출물 제출 전 확인:

- [ ] 시장 규모가 수치로 제시되었는가?
- [ ] 최소 3개 이상의 경쟁사가 분석되었는가?
- [ ] 경쟁사 분석에 실제 URL과 스크린샷이 포함되었는가?
- [ ] 고객 Pain Point가 구체적인 상황과 함께 기술되었는가?
- [ ] 페르소나에 실제 목소리(인용구)가 포함되었는가?
- [ ] 전략적 시사점이 다음 단계(Planning)에 actionable한가?

## Tools Used

- `web_search`: 시장 데이터, 트렌드 검색
- `web_fetch`: 경쟁사 서비스 페이지 분석
- `view`: 사용자 제공 레퍼런스 문서 확인
