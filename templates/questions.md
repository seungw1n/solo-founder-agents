# 구체화 질문 템플릿

> Orchestrator가 짧은 아이디어를 PRD로 변환하기 위해 던지는 질문들

---

## 사용 원칙

1. **모든 질문을 한 번에 던지지 않는다**
   - 필수 질문 먼저 → 선택 질문은 맥락에 따라

2. **답변 형식을 유도한다**
   - 선택지 제공 또는 예시 제시

3. **왜 물어보는지 설명한다**
   - 질문의 목적을 간단히 언급

---

## 필수 질문 (Always Ask)

### Q1: 궁극적인 목표 🎯

> "이 프로젝트의 궁극적인 목표는 무엇인가요?"

**선택지:**
```
a) PMF 탐색 - 새 제품의 시장 적합성 확인
b) 가설 검증 - 특정 아이디어/기능의 유효성 실험
c) 기능 확장 - 기존 제품에 새 기능 추가
d) 리브랜딩 - 브랜드 이미지/경험 재정립
e) 기타 (직접 입력)
```

**왜 물어보는가:**
- PRD 템플릿 선택
- 워크플로우 결정 (어떤 에이전트를 어떤 순서로)

---

### Q2: 성공의 정의 📊

> "이 프로젝트가 성공하면 어떤 변화가 일어나나요?"

**답변 가이드:**
```
1. 사용자에게: (예: "전세사기 걱정 없이 계약할 수 있다")
2. 비즈니스에: (예: "월 1,000명 이상 사용")
3. 어떻게 측정: (예: "체크 완료 후 '유용했다' 평가 80%+")
```

**왜 물어보는가:**
- North Star Metric 정의
- 성공 기준 설정

---

### Q3: 타겟 사용자 👤

> "정확히 누구를 위한 서비스인가요?"

**답변 가이드:**
```
1. 인구통계: (예: "20대 후반~30대 초반")
2. 상황/맥락: (예: "첫 전세 계약을 앞둔 사회초년생")
3. 현재 대안: (예: "부동산 커뮤니티에서 정보 수집")
```

**추가 질문 (필요시):**
- "왜 이 타겟인가요? 다른 그룹은 고려했나요?"
- "이 타겟에게 직접 이야기해본 적 있나요?"

**왜 물어보는가:**
- 페르소나 정의
- Research 범위 설정

---

## 상황별 선택 질문

### Q4: 산출물의 목표와 형태 📄
> "이번 프로젝트에서 가장 중요한 산출물은 무엇인가요?"

**선택지 (복수 선택 가능):**
```
a) 랜딩 페이지 - 가치 제안 검증
b) MVP 앱/웹 - 핵심 기능 검증
c) 프로토타입 - 사용성 검증
d) GTM 전략 - 시장 진입 계획
e) 데이터 분석 - 현황 파악
f) 기타: ___
```

**왜 물어보는가:**
- 에이전트 우선순위 결정
- 산출물 형식 결정

---

### Q5: 필수 에이전트/단계 🔧
> "꼭 거쳐야 할 단계나 특별히 중요한 에이전트가 있나요?"

**예시:**
```
- "경쟁사 분석이 특히 중요해요"
- "브랜딩/스토리텔링에 신경 써주세요"
- "디자인은 Figma MCP로 해주세요"
- "데이터 분석부터 시작해야 해요"
```

**왜 물어보는가:**
- 워크플로우 커스터마이징
- 에이전트 깊이 조절

---

### Q6: 레퍼런스 📚
> "참고할 자료가 있나요?"

**유형:**
```
1. URL: 경쟁사, 벤치마크 서비스
2. 문서: 기존 기획서, 데이터, 리서치
3. 이미지: UI 참고, 무드보드
```

**왜 물어보는가:**
- Research 방향 설정
- Design 톤앤매너 결정

---

### Q7: 제약 조건 ⚠️
> "고려해야 할 제약이 있나요?"

**유형:**
```
1. 기간: (예: "2주 내 MVP 필요")
2. 기술: (예: "React + Supabase만 사용")
3. 예산: (예: "월 $50 이내로 운영")
4. 기타: (예: "특정 디자인 시스템 사용 필수")
```

**왜 물어보는가:**
- 범위 조정
- 기술 스택 결정

## 목표별 추가 질문

### PMF 탐색 시 추가 질문

**Q-PMF-1: Aha Moment**
> "사용자가 '이거다!' 하고 느낄 순간은 언제일까요?"

예시: "주소 입력 30초 만에 위험도 결과를 보는 순간"

**Q-PMF-2: 초기 채널**
> "첫 100명 사용자를 어떻게 모을 계획인가요?"

예시: "부동산 커뮤니티", "지인 네트워크", "SNS"

---

### 가설 검증 시 추가 질문

**Q-EXP-1: 가설 명시**
> "검증하려는 가설을 정확히 표현해주세요."

형식: "만약 [X]를 하면, [Y]가 [Z]만큼 변할 것이다."

**Q-EXP-2: 현재 데이터**
> "현재 baseline 데이터가 있나요?"

---

### 기능 확장 시 추가 질문

**Q-FEAT-1: 배경 데이터**
> "이 기능이 필요하다고 판단한 데이터/피드백이 있나요?"

**Q-FEAT-2: 기존 시스템**
> "기존 제품의 기술 스택과 제약사항을 알려주세요."

---

### 브랜드 중심 서비스 시 추가 질문

**Q-BRAND-1: 브랜드 성격**
> "이 서비스의 성격을 형용사 3개로 표현하면?"

예시: "신뢰감 있는, 전문적인, 친근한"

**Q-BRAND-2: 경쟁 포지셔닝**
> "경쟁사 대비 어떤 이미지로 기억되고 싶나요?"

---

## 질문 우선순위 결정 로직

```
START
  │
  ├─ Q1: 목표 (필수)
  │
  ├─ Q2: 성공 정의 (필수)
  │
  ├─ Q3: 타겟 (필수)
  │
  └─ 목표에 따라 분기
       │
       ├─ PMF 탐색 → Q4, Q5, Q6, Q-PMF-1, Q-PMF-2
       │
       ├─ 가설 검증 → Q-EXP-1, Q-EXP-2, Q7
       │
       ├─ 기능 확장 → Q-FEAT-1, Q-FEAT-2, Q4
       │
       └─ 브랜드 중심 → Q-BRAND-1, Q-BRAND-2, Q5, Q6
```

---

## 질문 후 확인

**질문 완료 후:**
```
"질문은 여기까지입니다. 
제가 이해한 내용을 정리해볼게요:

[요약]
- 목표: {{goal}}
- 타겟: {{target}}
- 성공 기준: {{success_criteria}}
- 필수 단계: {{required_stages}}
- 제약: {{constraints}}

맞나요? 수정할 부분이 있으면 알려주세요.
확인되면 PRD를 생성하고 첫 번째 단계를 시작하겠습니다."
```
